구현 결과를 체크리스트로 반환할 것.

핵심요구사항) 다중 이미지 업로드 시 순차 업로드 대신 Promise.all을 활용하여 동시에 업로드하도록 개선할 것.

핵심요구사항) 사용자가 여러 이미지를 빠르게 선택하거나, 여러 이미지를 한 번에 선택할 수 있는 경우, 모든 이미지를 병렬로 업로드하여 속도를 개선할 것.

핵심요구사항) 기존의 개별 이미지 업로드 기능은 유지하되, 다중 이미지 업로드 시에만 Promise.all을 활용할 것.

구현방법) 
1. onChangeFile 함수를 수정하여 다중 파일 선택을 지원하도록 변경
2. 선택된 모든 파일을 배열로 수집
3. 각 파일에 대해 업로드 Promise를 생성
4. Promise.all을 사용하여 모든 업로드를 동시에 실행
5. 모든 업로드가 완료되면 결과를 images 배열에 반영

주의사항) 
- 기존의 단일 이미지 업로드 기능(index별 업로드)은 유지해야 함
- 파일 유효성 검사는 각 파일에 대해 수행해야 함
- 업로드 실패 시 적절한 에러 처리가 필요함
- 일부 이미지 업로드가 실패해도 성공한 이미지는 반영되어야 함
- 업로드 중 상태 표시를 고려할 수 있음 (선택사항)

예시코드구조:
```typescript
// 방법 1: 다중 파일 선택 지원 (input의 multiple 속성 활용)
const onChangeFile = async (event: React.ChangeEvent<HTMLInputElement>, startIndex: number) => {
  const files = event.target.files;
  if (!files || files.length === 0) return;

  // 모든 파일에 대한 유효성 검사
  const validFiles: File[] = [];
  for (let i = 0; i < files.length; i++) {
    const file = files[i];
    const isValid = checkValidationFile(file);
    if (isValid) {
      validFiles.push(file);
    }
  }

  if (validFiles.length === 0) return;

  try {
    // 모든 파일에 대한 업로드 Promise 생성
    const uploadPromises = validFiles.map((file) =>
      uploadFile({ variables: { file } })
    );

    // Promise.all을 사용하여 모든 업로드를 동시에 실행
    const results = await Promise.all(uploadPromises);

    // 업로드된 이미지 URL들을 images 배열에 반영
    const newImages = [...images];
    results.forEach((result, index) => {
      const targetIndex = startIndex + index;
      if (targetIndex < newImages.length) {
        newImages[targetIndex] = result.data?.uploadFile.url || "";
      }
    });
    setImages(newImages);
  } catch (e) {
    console.error(e);
    alert("일부 이미지 업로드에 실패했습니다.");
  }
};

// 방법 2: 기존 단일 업로드 유지 + 다중 업로드 헬퍼 함수 추가
const uploadMultipleFiles = async (files: File[], startIndex: number) => {
  const validFiles = files.filter(file => checkValidationFile(file));
  if (validFiles.length === 0) return;

  try {
    const uploadPromises = validFiles.map((file) =>
      uploadFile({ variables: { file } })
    );

    const results = await Promise.all(uploadPromises);
    const newImages = [...images];
    
    results.forEach((result, index) => {
      const targetIndex = startIndex + index;
      if (targetIndex < newImages.length && result.data?.uploadFile.url) {
        newImages[targetIndex] = result.data.uploadFile.url;
      }
    });
    
    setImages(newImages);
  } catch (e) {
    console.error(e);
    // 부분 실패 처리: 성공한 것만 반영
    alert("일부 이미지 업로드에 실패했습니다.");
  }
};

const onChangeFile = async (event: React.ChangeEvent<HTMLInputElement>, index: number) => {
  const files = event.target.files;
  if (!files || files.length === 0) return;

  // 단일 파일 선택인 경우 기존 로직 유지
  if (files.length === 1) {
    const file = files[0];
    const isValid = checkValidationFile(file);
    if (!isValid) return;

    try {
      const result = await uploadFile({ variables: { file } });
      const newImages = [...images];
      newImages[index] = result.data?.uploadFile.url || "";
      setImages(newImages);
    } catch (e) {
      console.error(e);
      alert("이미지 업로드에 실패했습니다.");
    }
    return;
  }

  // 다중 파일 선택인 경우 Promise.all 활용
  await uploadMultipleFiles(Array.from(files), index);
};
```

- 이미지 업로드 개수 제한을 고려할 수 있음 (현재는 최대 3개)
